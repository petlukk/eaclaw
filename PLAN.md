# eaclaw — Cache-Resident SIMD Agent Framework

## Context

Build a multi-agent framework inspired by ironclaw but designed for extreme performance. The hot path (everything before the LLM call) runs through Eä SIMD kernels that fit in L1 cache. Rust handles I/O, async, and LLM orchestration. The result: a chatbot pipeline where input processing runs at memory bandwidth (~GB/s) instead of scalar string-matching speed.

**Key principle:** SIMD filter + scalar verify. Eä kernels rapidly identify candidate positions (rejecting >99% of bytes). Cheap Rust scalar code verifies matches at candidates only.

## Phase 1: Eä Kernels (build first)

### Step 1: `byte_classifier.ea`

Classify every byte: whitespace(1), letter(2), digit(4), punctuation(8), non-ASCII(16).

- **Reference:** `/root/dev/eacompute/demo/tokenizer_prepass/prepass.ea` (`classify_u8x16`)
- **Types:** `u8x16`
- **Intrinsics:** `load`, `store`, `splat`, `select`, `.==`, `.>=`, `.<=`, `.|`
- **Export:** `classify_bytes(text: *u8, flags: *mut u8, len: i32)`
- **Cache:** ~1 KB instructions, 0 bytes working data. Single streaming pass.

### Step 2: `json_scanner.ea`

Find structural JSON characters: `{ } [ ] : , " \`. Two functions:

1. `count_json_structural(text: *u8, len: i32, out_count: *mut i32)` — count for pre-allocation
2. `extract_json_structural(text: *u8, len: i32, out_pos: *mut i32, out_types: *mut u8, out_count: *mut i32)` — extract positions via `movemask` + bit loop

- **Reference:** `/root/dev/eacompute/demo/1brc/kernels/scan.ea` (movemask + bit extraction)
- **Types:** `u8x16` (not u8x32 — movemask sign bit issue)
- **Intrinsics:** `load`, `splat`, `select`, `.==`, `.|`, `movemask`
- **Note:** Quote tracking (in-string state) is scalar — sequential dependency. SIMD finds candidates, scalar filters.

### Step 3: `leak_scanner.ea`

SIMD two-byte prefix filter for secret patterns (sk-, AKIA, ghp_, xox, SG., sess_, -----BEGIN, etc.)

**Bitmask output (not position arrays)** — return a `u16` mask per 16-byte block. Rust processes the masks with `tzcnt` + `mask &= mask - 1` bit loop. Fewer memory writes, better cache locality, simpler kernel.

- **Export:** `scan_leak_prefixes(text: *u8, len: i32, out_masks: *mut i32, out_n_blocks: *mut i32)`
- **Algorithm:** For each 16-byte chunk, compare every byte against the set of "interesting first bytes" using chained `select` + `.|`. `movemask` → store one `i32` mask per block.
- **Rust side:** iterate blocks, for each non-zero mask: `while mask != 0 { bit = tzcnt(mask); pos = block*16 + bit; verify(pos); mask &= mask - 1; }`
- **Cache:** ~1.5 KB instructions.

### Step 4: `sanitizer.ea`

Same bitmask architecture as leak_scanner but for injection pattern first bytes (i, y, s, <, |, [, etc.)

- **Export:** `scan_injection_prefixes(text: *u8, len: i32, out_masks: *mut i32, out_n_blocks: *mut i32)`

### Step 5: `command_router.ea`

**Hash-based command matching.** Read first 4 bytes as a u32-equivalent, compare against known command hashes. Almost free — a few comparisons.

- **Export:** `match_command(text: *u8, len: i32, out_match: *mut i32)`
- **Algorithm:** Load first 4 bytes, construct a u32 hash from them. Match against precomputed hashes: `/hel` → help, `/qui` → quit, `/too` → tools, etc. Fall through to -1 (no match).
- **Cache:** ~0.5 KB instructions. Likely runs entirely in registers.

### Step 6: `build.sh`

```bash
#!/bin/bash
set -euo pipefail
KERNEL_DIR=kernels
OUT_DIR=target/kernels

for f in $KERNEL_DIR/*.ea; do
    stem=$(basename "$f" .ea)
    ea "$f" --lib -o "$OUT_DIR/lib${stem}.so"
    ea bind "$f" --rust
done
```

## Phase 3: Rust Agent Framework

### Step 9: Foundation (`error.rs`, `config.rs`)

`Config` reads from env:

- `ANTHROPIC_MODEL` — default `claude-sonnet-4-20250514`

- `COMMAND_PREFIX` — default `/`

### Step 10: Safety layer (`safety/`)

The novel integration. `SafetyLayer` pre-allocates buffers, calls Eä kernels:

```
scan_input(content) → {
    1. classify_bytes (Eä) → byte flags
    2. scan_injection_prefixes (Eä) → bitmask per 16-byte block
    3. Rust bit-loop: tzcnt + verify at each set bit position
    4. scan_leak_prefixes (Eä) → bitmask per 16-byte block
    5. Rust bit-loop: tzcnt + verify leak patterns
    → ScanResult { injection_found, leaks_found, details }
}
```

Pattern lists sourced from ironclaw:
- `sanitizer.rs`: 16 injection strings from `/root/dev/ironclaw/src/safety/sanitizer.rs`
- `leak_detector.rs`: 15+ secret prefixes from `/root/dev/ironclaw/src/safety/leak_detector.rs`

### Step 11: Channel (`channel/repl.rs`)

Minimal REPL with rustyline. Channel trait:
```rust
#[async_trait]
pub trait Channel: Send + Sync {
    fn name(&self) -> &str;
    async fn recv(&self) -> Option<String>;
    async fn send(&self, content: &str);
}
```

### Step 12: LLM (`llm/anthropic.rs`)

Direct Anthropic Messages API via reqwest:
- POST `https://api.anthropic.com/v1/messages`
- Headers: `x-api-key`, `anthropic-version: 2023-06-01`
- Tool use support (tools array in request, tool_use/tool_result content blocks)
- Streaming optional for MVP (non-streaming first)

### Step 13: Tools (`tools/`)

5 tools with JSON schema definitions:
- `echo` — return input (test tool)
- `time` — current UTC time
- `shell` — execute command, return stdout/stderr
- `http` — GET a URL, return body
- `memory` — in-memory key-value store (write/read/list)

### Step 14: Agent loop (`agent/mod.rs`)

```
loop {
    msg = channel.recv()
    if starts_with(prefix) → handle_command (quit, help, tools)
    else →
        safety.scan_input(msg)
        messages.push(user)
        loop {  // agentic tool loop
            response = llm.chat(messages, tools)
            if tool_use blocks →
                for each tool_use: execute, safety.scan_output(result), push tool_result
            else →
                channel.send(text)
                break
        }
}
```

- Messages with injection patterns → detected
- Messages with API key patterns → detected
- Clean messages → no false positives
- Large tool output (100 KB) → scanned correctly

### Step 17: Criterion benchmarks (`benches/safety_bench.rs`)

Compare Eä kernels vs:
- `aho_corasick` crate (add as dev-dependency only)
- Scalar Rust loops
- On 1 KB, 10 KB, 100 KB inputs

Report throughput as:
- **GB/s** (primary metric — how close to memory bandwidth?)
- **cycles/byte** (how much compute per byte?)
- **branch mispredicts** (via `perf stat` — should be near zero for SIMD path)

## Key Design Decisions

1. **SIMD filter + scalar verify** — Eä finds candidates, Rust verifies. Simpler kernels, same end result.
2. **Separate kernels, not fused** — Each fits in L1 independently. Input data (<100 KB) also fits in L1. Fusion would add instructions without saving memory traffic.
3. **u8x16 not u8x32 for movemask** — Avoids sign-bit issue documented in eacompute's 1BRC scan.ea.
4. **Pre-built .so files** — No ea compiler dependency at `cargo build` time. `build.sh` for regeneration.
5. **No database for MVP** — In-memory context only. Add libSQL later.
6. **No regex/aho-corasick deps** — The whole point: Eä replaces these in the hot path.
7. **Anthropic Claude as primary LLM** — Direct reqwest, no SDK dependency.
